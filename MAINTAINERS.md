## Maintainer Responsibilities

### Core Maintainers

Core maintainers have full commit access and are responsible for:

1. **Code Review** - Reviewing and merging pull requests
2. **Release Management** - Creating releases and managing versions
3. **Issue Triage** - Labeling, prioritizing, and assigning issues
4. **Community Management** - Enforcing Code of Conduct, welcoming contributors
5. **Documentation** - Ensuring docs stay current and comprehensive
6. **Security** - Responding to security reports, creating patches
7. **Architecture** - Making major architectural decisions
8. **Dependencies** - Managing dependency updates and audits

## Becoming a Maintainer

We welcome contributors who demonstrate:

1. **Consistent Contributions** - Regular, high-quality PRs over 3+ months
2. **Code Review Participation** - Helpful, constructive reviews of others' work
3. **Community Engagement** - Answering questions, helping new contributors
4. **Technical Excellence** - Deep understanding of project architecture
5. **Reliability** - Following through on commitments
6. **Communication** - Clear, respectful, timely communication
7. **Alignment** - Agreement with project values and direction

### Path to Maintainership

1. **Contributor** - Make your first PR, get familiar with codebase
2. **Regular Contributor** - 5+ merged PRs, participate in discussions
3. **Trusted Contributor** - 15+ merged PRs, code reviews, issue triage help
4. **Maintainer Nomination** - Nominated by existing maintainer
5. **Maintainer Vote** - Approved by majority of core team
6. **Onboarding** - Access granted, responsibilities explained
7. **Announcement** - Welcome to the team!

## Maintainer Onboarding

New maintainers receive:

1. **Repository Access** - Commit access to relevant repos
2. **Mentorship** - Paired with experienced maintainer for 30 days
3. **Documentation** - Access to maintainer guides and processes
4. **Communication Channels** - Added to maintainer chat/email
5. **Release Access** - npm publish access (after 90 days)
6. **Security Access** - Added to security@supranorma.dev

## Maintainer Emeritus

Maintainers may step down to emeritus status if:

- Life circumstances change (welcomed back anytime!)
- Sustained inactivity (6+ months)
- Shifting focus to other projects

Emeritus maintainers:
- Keep their historical contributions and recognition
- May return to active status anytime
- Remain welcome in the community

## Conflict Resolution

If maintainers disagree:

1. **Discussion** - Talk it through, understand perspectives
2. **Documentation** - Write down pros/cons of each option
3. **Vote** - Simple majority of core maintainers (if needed)
4. **Escalation** - Project lead makes final decision (rare)
5. **Document Decision** - Record rationale for future reference

## Maintainer Communication

- **Weekly Sync** - Optional async check-in (GitHub Discussions)
- **Monthly Meeting** - Optional video call for major topics
- **Emergency Channel** - For security issues and critical bugs
- **Public Discussion** - Most decisions made in public issues/PRs

## Code of Conduct

All maintainers must:

- Follow the [Code of Conduct](CODE_OF_CONDUCT.md)
- Model excellent behavior for the community
- Enforce CoC fairly and consistently
- Report CoC violations promptly
- Maintain confidentiality of reports

## Time Commitment

Expected time commitment (flexible, understanding real life happens):

- **Code Review**: 2-5 hours/week
- **Issue Triage**: 1-2 hours/week
- **Community Engagement**: 1-2 hours/week
- **Release Management**: 2-4 hours/month (when responsible)
- **Total**: ~4-10 hours/week average

## Maintainer Benefits

While this is volunteer work, maintainers gain:

- **Skills** - Experience in project leadership, mentorship
- **Network** - Connections with other contributors and maintainers
- **Resume** - Notable open-source project on CV
- **Speaking** - Opportunities for conference talks
- **Early Access** - First to see new features and directions
- **Impact** - Shape a project used by others

## Stepping Down

If you need to step down:

1. **Notice** - Let core team know (1 month preferred, not required)
2. **Handoff** - Document any pending work or responsibilities
3. **Access Removal** - We'll remove commit access, keep you in emeritus
4. **Thank You** - Sincere gratitude for your contributions!
5. **Return Path** - Door is always open to return
